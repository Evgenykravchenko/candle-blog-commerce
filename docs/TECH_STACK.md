# Стек Технологий: Candle Blog Commerce

В этом документе описан весь технологический стек проекта, используемые библиотеки и обоснование их выбора.

## 1. Core (Ядро)

- **Next.js 16 (App Router)**
  - _Зачем:_ Современный фреймворк для React. Обеспечивает лучший SEO (благодаря SSR/SSG), быструю маршрутизацию и оптимизацию изображений/шрифтов из коробки. Версия 16 предоставляет новейшие возможности производительности.
- **React 19**
  - _Зачем:_ Библиотека для построения пользовательских интерфейсов. Версия 19 (в связке с Next.js) обеспечивает поддержку серверных компонентов (RSC) и улучшенную работу с состоянием.
- **TypeScript**
  - _Зачем:_ Типизированный JavaScript. Помогает избегать ошибок на этапе разработки, улучшает автодополнение в IDE и делает код более поддерживаемым при масштабировании проекта.

## 2. Стилизация и UI

- **Tailwind CSS 4**
  - _Зачем:_ Утилитарный CSS-фреймворк. Позволяет верстать быстро, прямо в HTML, гарантирует отсутствие конфликтов стилей и минимальный размер итогового CSS файла. 4-я версия работает еще быстрее.
- **Lucide React**
  - _Зачем:_ Набор красивых, консистентных и легких SVG иконок. Стандарт де-факто для современных React проектов "чистого" дизайна.
- **clsx & tailwind-merge & class-variance-authority (CVA)**
  - _Зачем:_ Утилиты для условного объединения классов CSS. Необходимы для создания переиспользуемых компонентов (кнопок, карточек) с разными вариантами (vriants) без конфликтов стилей Tailwind.

## 3. Анимации

- **Framer Motion** (Планируется к установке)
  - _Зачем:_ Мощная библиотека для декларативных анимаций в React. Позволяет легко создавать сложные переходы (layout animations), анимации при скролле и микровзаимодействия, которые критически важны для премиум-ощущения от сайта.
- **Lenis Scroll** (Планируется)
  - _Зачем:_ Библиотека для плавного (smooth) скролла. Убирает "рывки" стандартного браузерного скролла, делая навигацию по лендингу "масляной" и дорогой.

## 4. Качество кода и Конфигурация

- **ESLint**
  - _Зачем:_ Статический анализатор кода. Находит ошибки и следит за соблюдением единого стиля кода.
- **Prettier** (Рекомендуется добавить)
  - _Зачем:_ Автоматическое форматирование кода. Избавляет от споров о запятых и отступах.

## 5. Планы на будущее (Интеграции)

- **WordPress (Headless)**
  - _Зачем:_ Удобная админка для заказчика. Будет использоваться только как API для отдачи контента (статей блога, товаров).
- **WPGraphQL / ACF**
  - _Зачем:_ Для получения структурированных данных из WordPress в Next.js.
