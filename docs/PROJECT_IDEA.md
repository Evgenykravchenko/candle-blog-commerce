# Дизайн-документ и Требования к Проекту: Candle Blog Commerce

## 1. Обзор проекта

Лендинг премиум-класса для бренда свечей ручной работы.
**Главные цели:**

1.  Эффектная презентация продукта, создающая "вау-эффект" и ощущение премиальности.
2.  Перенаправление трафика на маркетплейс Ozon для совершения покупки.
3.  Привлечение органического трафика через SEO-оптимизированный блог.
4.  В перспективе: легкое управление контентом через CMS (Headless WordPress).

## 2. Дизайн и Эстетика

- **Стиль:** Минимализм, "тихая роскошь" (Quiet Luxury), скандинавский или джапанди стиль.
- **Цветовая палитра:** Естественные, "дорогие" оттенки:
  - _Основной:_ Теплый белый, кремовый, слоновая кость.
  - _Акцентный:_ Терракотовый, глубокий оливковый, шалфейный, теплый угольный.
  - _Текст:_ Не чисто черный, а глубокий графит или темно-коричневый для мягкости.
- **Типографика:** Сочетание элегантного Serif (с засечками) для заголовков (например, Playfair Display, Cinzel) и чистого Sans-Serif (без засечек) для основного текста (Inter, Lato, Montserrat).
- **Визуал:** Много "воздуха" (отступов), качественные атмосферные фото, видео-фоны.
- **Анимации:**
  - Плавный скролл (Lenis Scroll).
  - Параллакс-эффекты для изображений.
  - Fade-in появление элементов при скролле.
  - Интерактивные карточки товаров (смена фото/видео при наведении).
  - Микро-анимации кнопок и ссылок.

## 3. Структура Лендинга (Секции)

### 1. Hero Section (Главный экран)

- **Контент:** Полноэкранное атмосферное видео (горение свечи, уютный интерьер) или слайдер.
- **Элементы:**
  - Крупный заголовок с УТП (Уникальное Торговое Предложение).
  - Слоган, передающий настроение.
  - Кнопка CTA (Call to Action) "Выбрать аромат" -> скролл к каталогу.

### 2. О Бренде (Философия)

- **Контент:** История создателя, философия ручного труда.
- **Акцент:** Натуральность материалов (соевый/кокосовый воск, премиальные отдушки), экологичность.
- **Визуал:** Фото мастерской, рук мастера, процесса заливки. Эстетичная инфографика преимуществ.

### 3. Витрина Свечей (Интерактивный Каталог)

- **Функционал:** Не просто списком, а красивая сетка или горизонтальный скролл.
- **Фильтры/Табы:** По коллекциям (Осень, Зима, Relax, Energy).
- **Карточка товара:**
  - Качественное фото.
  - Название свечи.
  - **Пирамида аромата:** Верхние, средние, базовые ноты (визуализировано иконками или списком).
  - Цена.
  - Яркая кнопка **"Купить на Ozon"** (открывает новую вкладку).
- **Интерактив:** При наведении/клике — всплывающее видео-превью или детальное описание настроения аромата.

### 4. Секция-Конструктор "Подбери настроение" (Опционально)

- Простой интерактив: "Для чего вы ищете свечу?" (Для ванны / Для ужина / Для работы) -> Система предлагает подходящие ароматы.

### 5. Отзывы (Social Proof)

- Стильный слайдер с реальными фото свечей в интерьерах клиентов.
- Текст отзыва и ссылка на источник (например, скрин с Ozon или Instagram).

### 6. Блог (SEO-драйвер)

- Сетка последних 3-х статей.
- Стильные превью и интригующие заголовки ("Как правильно жечь свечи?", "Топ-5 ароматов для осени").
- Кнопка "Читать блог" -> переход на страницу `/blog`.

### 7. Контакты и Где нас найти

- Ссылки на соцсети (Instagram, Telegram, Pinterest).
- Форма подписки на рассылку (новинки, скидки).
- Адрес (если уместно) или карта точек продаж.
- Финальный CTA "Перейти в магазин на Ozon".

### 8. Footer (Подвал)

- Логотип.
- Навигация по сайту.
- Публичная оферта / Политика конфиденциальности.
- Copyright.

## 4. Блог-система

- Отдельная страница `/blog` со списком всех статей.
- Категории статей (Советы, Декор, Ароматерапия).
- Страница статьи:
  - Красивая верстка текста.
  - Интеграция товаров внутрь статьи ("В этой статье мы говорили про свечу X — [Купить]").
  - Шеринг в соцсети.
- **SEO:** Meta tags (Title, Description), Open Graph, Schema.org разметка, генерация Sitemap.xml.

## 5. Технический Стек

- **Framework:** **Next.js 15+ (App Router)** — для лучшего SEO, производительности и статической генерации (SSG/ISR).
- **Language:** **TypeScript** — строгая типизация для надежности.
- **Styling:** **Tailwind CSS 4** — скорость разработки и гибкость.
- **Animation:** **Framer Motion** — стандарт де-факто для React анимаций.
- **CMS (Headless):** **WordPress** (на будущее).
  - Используется API (REST или GraphQL) для получения контента.
  - Позволяет заказчику редактировать тексты, товары и блог через привычную админку WP, не касаясь кода.

## 6. План интеграции CMS (WordPress)

1.  Развернуть WordPress на хостинге.
2.  Установить плагины: ACF (Advanced Custom Fields) для полей товара, WPGraphQL (опционально).
3.  Создать Custom Post Types: "Candles" (Товары).
4.  Настроить Next.js на получение данных из WP при сборке (ISR - Incremental Static Regeneration), чтобы сайт летал как статика, но обновлялся при изменении контента.
