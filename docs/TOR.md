# Техническое Задание (ТЗ) на разработку веб-ресурса "Candle Blog Commerce"

## 1. Введение

### 1.1. Общие сведения

Проект представляет собой имиджевый промо-сайт (лендинг) с функциональностью блога для бренда свечей ручной работы. Основная цель — презентация продукции премиального качества, повышение лояльности через контент и перенаправление пользователей на маркетплейс Ozon для покупки.

### 1.2. Термины и определения

- **Сайт/Ресурс** — разрабатываемое веб-приложение.
- **CMS (Content Management System)** — система управления контентом (в перспективе Headless WordPress).
- **Маркетплейс** — внешняя торговая площадка Ozon.ru.
- **CTA (Call To Action)** — призыв к действию (кнопка "Купить", "Подписаться").

## 2. Цели и задачи проекта

### 2.1. Бизнес-цели

1.  Увеличить продажи свечей на Ozon за счет качественной презентации продукта.
2.  Повысить узнаваемость бренда и сформировать имидж "тихой роскоши".
3.  Создать канал привлечения органического трафика через SEO-статьи в блоге.
4.  Сформировать базу лояльных клиентов (через подписку на соцсети/рассылку).

### 2.2. Технические задачи

1.  Обеспечить высокую скорость загрузки (PageSpeed Green Zone > 90).
2.  Реализовать адаптивную верстку (Desktop, Tablet, Mobile).
3.  Обеспечить полную SEO-оптимизацию (SSR, мета-теги, Open Graph, микроразметка).
4.  Подготовить архитектуру для легкого подключения CMS в будущем.

## 3. Целевая аудитория

- **Пол:** Преимущественно женщины (80%), мужчины (20% - в поиске подарка).
- **Возраст:** 20–45 лет.
- **Интересы:** Уют, декор дома, хюгге, медитация, саморазвитие, эстетика, экологичность.
- **Потребности:** Создание атмосферы, снятие стресса, поиск эстетичного подарка.

## 4. Требования к Дизайну и UX/UI

### 4.1. Общая стилистика

- **Стиль:** Минимализм, "Quiet Luxury", премиальность, чистота.
- **Цветовая гамма:** Натуральные, пастельные тона (кремовый, бежевый, шалфей, терракота). Отсутствие "кричащих" цветов.
- **Типографика:** Сочетание акцидентного шрифта с засечками (Antiqua) для заголовков и гротеска (Sans-serif) для основного текста.
- **Воздух:** Большие отступы, отсутствие визуального шума.

### 4.2. Анимация и Интерактив

- **Плавный скролл:** Использование Lenis или аналогов для ощущения "тягучести" и плавности.
- **Микро-взаимодействия:** Плавные ховер-эффекты на кнопках, ссылках и карточках (изменение прозрачности, скейл, смена цвета).
- **Появление контента:** Fade-in анимация блоков при скролле.
- **Параллакс:** Легкий параллакс эффект на фоновых изображениях.

## 5. Функциональные требования по разделам

### 5.1. Header (Шапка сайта)

- **Логотип:** По центру или слева. Ссылка на главную.
- **Меню:** Ссылки-якоря (Каталог, О нас, Блог, Контакты).
- **Мобильная версия:** Меню-"гамбургер" с плавной анимацией открытия.
- **Поведение:** "Липкая" шапка (sticky) или скрытие при скролле вниз и появление при скролле вверх. Прозрачный фон на первом экране, плашмя при скролле.

### 5.2. Секция 1: Hero (Главный экран)

- **Фон:** Полноэкранное видео (loop) высокого качества (горение фитиля, дымок, текстура воска) или слайдер атмосферных фото.
- **Контент:**
  - Заголовок H1: Эмоциональный, передающий суть (напр. "Свет, создающий атмосферу").
  - Подзаголовок: УТП (напр. "Свечи из 100% соевого воска с нишевыми ароматами").
  - Кнопка CTA: "Выбрать свой аромат" (скролл к каталогу).

### 5.3. Секция 2: О бренде (Storytelling)

- **Макет:** Асимметричная сетка (Текст + Фото).
- **Контент:** История создания, ценности (ручная работа, экологичность, безопасность).
- **Медиа:** Фото/видео процесса производства (заливка, упаковка).

### 5.4. Секция 3: Интерактивный Каталог (Shop window)

- **Структура:** Сетка товаров (Grid) или горизонтальный скролл (Slider).
- **Фильтрация:** Табы по коллекциям (Например: "Relax", "Energy", "Romantic").
- **Карточка товара:**
  - Фото (при наведении смена на фото в интерьере или видео).
  - Название свечи.
  - Объем / Время горения.
  - Цена.
  - Кнопка "Купить на Ozon" (открытие внешней ссылки в новой вкладке `target="_blank"`).
- **Детальный просмотр (Modal/Pop-up или разворот):** (Опционально)
  - Описание пирамиды аромата (Верхние, Сердце, База).
  - Состав.

### 5.5. Секция 4: "Подбор аромата" (Quiz/Constructor) - Опционально

- Простой интерактивный блок с 2-3 вопросами (Для кого? Какое настроение?).
- Результат: Рекомендация 1-2 свечей с кнопкой покупки.

### 5.6. Секция 5: Блог (Превью)

- Вывод последних 3-х статей.
- Карточка статьи: Фото, Заголовок, Краткое описание, Дата, Кнопка "Читать".
- Переход на страницу `/blog` для просмотра всех записей.

### 5.7. Секция 6: Отзывы (Social Proof)

- Слайдер отзывов.
- Формат: Фото клиента/товара + Текст + Имя + Ссылка на источник (скриншот с маркетплейса).

### 5.8. Секция 7: Контакты и Футер

- Ссылки на соцсети (TG, Inst, VK).
- Форма подписки на рассылку (Email).
- Навигация.
- Юридическая информация (Copyright, Оферта).

### 5.9. Страница Блога `/blog` (и `/blog/[slug]`)

- **Список:** Сетка статей с пагинацией или кнопкой "Загрузить еще".
- **Статья:**
  - Красивая типографика, разбивка на абзацы.
  - Изображения, цитаты.
  - **Товарная врезка:** Возможность вставить карточку товара прямо в текст статьи (native advertising).
  - Блок "Поделиться" и "Похожие статьи".

## 6. Технические требования

### 6.1. Стек технологий

- **Frontend:** Next.js 16 (App Router), React 19.
- **Язык:** TypeScript.
- **Стили:** Tailwind CSS 4.
- **Анимации:** Framer Motion.
- **Пакетный менеджер:** npm/yarn/pnpm.

### 6.2. Производительность (Performance)

- Оптимизация изображений через компонент `next/image` (WebP/AVIF).
- Lazy loading для изображений ниже первого экрана.
- Code splitting (автоматически в Next.js).
- Минимизация CLS (Cumulative Layout Shift) — жесткие размеры для медиа-контейнеров.

### 6.3. SEO (Search Engine Optimization)

- Server-Side Rendering (SSR) или Static Site Generation (SSG) для всех страниц.
- Генерация `sitemap.xml` и `robots.txt`.
- Семантическая верстка (H1-H6, article, section, aside).
- Микроразметка Schema.org (Product, Article, Organization).
- Мета-теги (Title, Description) уникальны для каждой страницы.

### 6.4. Требования к CMS (Headless)

- Возможность создания и редактирования товаров (Название, Цена, Ссылка, Описание, Фото).
- Возможность ведения блога (WYSIWYG редактор для статей).
- Управление SEO-полями через CMS.

## 7. Этапы реализации

1.  **Подготовка:** Настройка репозитория, установка зависимостей, конфигурация линтеров и Tailwind.
2.  **Верстка UI Kit:** Создание базовых компонентов (кнопки, инпуты, типографика).
3.  **Разработка Лендинга:** Реализация секций главной страницы, анимаций, адаптива.
4.  **Разработка Блога:** Создание страниц списка и детальной страницы статьи (пока на мок-данных).
5.  **Интеграция:** Подготовка данных в JSON/TS файлах (в будущем замена на API CMS).
6.  **SEO и Оптимизация:** Настройка мета-тегов, проверка производительности.
7.  **Деплой:** Публикация на Vercel/VPS.

## 8. Критерии приемки

1.  Сайт корректно отображается в Chrome, Safari, Firefox, Edge (последние 2 версии).
2.  Сайт корректно отображается на мобильных устройствах (iOS, Android).
3.  Анимации работают плавно, не вызывают "фризов".
4.  Все ссылки "Купить" корректно ведут на соответствующие товары на Ozon.
5.  Страницы блога индексируются, имеют правильные мета-теги.
6.  Lighthouse Performance Score > 90 (Mobile & Desktop).
